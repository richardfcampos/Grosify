## Known limitations (phase 6)
- Signing out with pending mutations in the outbox loses the unsynced ones (clearLocalData wipes the queue). Warn/force sync before sign out in the future.

## Known limitations (phase 3)
- Session/membership require the API online: navigation with a fresh page-load offline falls back to login. Real usage (already logged in, goes offline mid-session via SPA nav) works — verified. Offline app shell via Workbox precache (build generates the SW; test in a prod build).
- Concurrent editing of the SAME item by 2 offline members: last-sync-wins (no per-field LWW). Rare at household scale.
- A mutation rejected with a received response (e.g., item_limit) is removed from the outbox but the local optimistic row remains — reconcile in the future.
